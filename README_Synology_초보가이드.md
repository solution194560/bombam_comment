# 시놀로지(Synology) Docker 완전 초보 가이드 — 봄밤 수집기

시놀로지를 처음 써도 따라 할 수 있게 단계별로 설명합니다.
순서: 0 개념 → 1 DSM 접속 → 2 모델 확인 → 3 Container Manager 설치 →
4 폴더/파일 올리기 → 5 로그인 쿠키 → 6 빌드·실행 → 7 결과 확인 → 8 자동화

---

## 0. 먼저 개념 (1분)
- **시놀로지 NAS** = 집/사무실에 두는 작은 리눅스 서버(저장장치).
- **DSM** = NAS를 조작하는 웹 화면. 브라우저로 들어갑니다 (윈도우 비슷한 GUI).
- **Container Manager** = DSM에서 Docker를 다루는 앱(예전 이름 "Docker").
- 우리가 할 일: 이 수집기를 Docker "컨테이너"로 NAS 안에서 돌리는 것.

---

## 1. DSM(웹 화면) 접속
1. NAS와 같은 네트워크(같은 공유기)에 PC를 연결.
2. 브라우저에서 다음 중 하나 입력:
   - `http://NAS이름:5000`  (예: `http://synology:5000`)
   - 또는 `http://NAS의IP주소:5000` (예: `http://192.168.0.10:5000`)
   - IP를 모르면: 윈도우는 `find.synology.com` 접속, 또는 공유기 관리페이지에서 확인.
3. 관리자 계정(admin 등)으로 로그인 → 윈도우 같은 바탕화면(DSM)이 나옵니다.

> DSM 버전이 **7.2 이상**이어야 아래 "프로젝트(빌드)" 기능이 있습니다.
> 좌상단 메뉴 → 제어판 → 정보 센터에서 DSM 버전 확인.

---

## 2. 내 NAS가 가능한 모델인지 확인 (중요)
이 프로그램은 크롬(Chromium)을 띄우므로 **CPU 종류와 메모리**가 중요합니다.
1. DSM → **제어판 → 정보 센터 → 일반** 탭
2. **CPU 모델**과 **메모리** 확인.
   - CPU가 **Intel / AMD / Realtek RTD129x / aarch64 계열** → 대부분 OK
   - CPU가 **아주 저가형 ARM(예: 일부 j 시리즈)** → 안 될 수 있음
   - 메모리는 **1GB 이상** 권장 (512MB는 위험)
3. 잘 모르겠으면 **NAS 모델명(예: DS220+, DS920+)** 을 저(클로드)에게 알려주세요. 가능 여부를 정확히 알려드릴게요.

---

## 3. Container Manager 설치
1. DSM → **패키지 센터**(Package Center) 열기
2. 검색창에 **Container Manager** 입력 → **설치**
3. 설치 후 좌측 메뉴/바탕화면에 **Container Manager** 아이콘이 생깁니다.

---

## 4. 소스 파일을 NAS에 올리기 (File Station, SSH 필요 없음)
1. DSM → **File Station** 열기 (NAS의 파일 탐색기)
2. 왼쪽에서 **docker** 라는 공유 폴더를 찾습니다.
   - 없으면: 제어판 → 공유 폴더 → 생성 → 이름 `docker`
3. `docker` 안에 폴더를 만듭니다: **bombam**
   - 그 안에 다시 **profile** , **work** 폴더 2개 생성.
   - 최종 구조:
     ```
     docker/bombam/
     docker/bombam/profile/   (로그인 쿠키 둘 곳 - 5번에서 채움)
     docker/bombam/work/      (결과 엑셀 나올 곳)
     ```
4. 내 PC에 있는 **이 작업 폴더의 파일 전부**를(아래 "꼭 필요한 파일" 참고)
   `docker/bombam/` 안으로 **드래그&드롭 업로드**.
   - File Station 창에 그냥 파일을 끌어다 놓으면 업로드됩니다.

### 꼭 필요한 파일 (이것들만 있으면 됨)
```
ridi_collector.py
1_전체수집.py
2_작품별_최신댓글.py
3_최근30일_댓글.py
account.json
requirements.txt
Dockerfile
docker-compose.yml
```
(debug*.py, page_dump.html, 기존 .xlsx 등 나머지는 안 올려도 됩니다.)

---

## 5. 로그인 쿠키 만들기 (가장 중요!)
성인 콘텐츠라 **로그인**이 필요한데, NAS는 화면이 없어 직접 로그인할 수 없습니다.
그래서 **내 맥에서 한 번만 로그인**하고, 그때 생긴 "쿠키 폴더"를 NAS에 올립니다.

### 5-1. 맥에서 로그인 1회
맥 터미널에서 이 작업 폴더로 이동 후:
```bash
HEADLESS=0 python3 2_작품별_최신댓글.py
```
- 크롬 창이 뜨면 **리디북스에 로그인** (성인인증까지) → 터미널에 뜬 안내대로 **Enter**.
- 이러면 맥의 홈에 숨김폴더 `~/.browser_data_dir` 가 생깁니다(=쿠키 저장소).

### 5-2. 쿠키 폴더를 NAS에 올리기
1. 맥 Finder에서 **Cmd + Shift + G** → `~/.browser_data_dir` 입력 → 이동
2. 그 폴더 **안의 내용 전체**를 복사
3. DSM **File Station → docker/bombam/profile/** 에 **드래그&드롭** 업로드
   (즉, profile 폴더 안에 쿠키 파일들이 들어가야 함)

> 나중에 쿠키가 만료(수 주~수 개월)되면 5번만 다시 하면 됩니다.

---

## 6. Container Manager로 빌드 & 실행
1. **Container Manager** 열기 → 왼쪽 **프로젝트(Project)** 클릭 → **생성**
2. 설정:
   - **프로젝트 이름**: `bombam`
   - **경로**: `/docker/bombam` 선택 (4번에서 만든 폴더)
   - **소스**: "기존 docker-compose.yml 사용"이 자동 인식됩니다.
3. **다음 → 완료**. 그러면 자동으로 **이미지 빌드 → 컨테이너 실행**이 시작됩니다.
   - 처음 빌드는 크롬 다운로드 때문에 **5~15분** 걸릴 수 있어요(인터넷 속도/모델 따라).
4. 빌드 로그/실행 로그는 같은 화면에서 볼 수 있습니다.

> 작업이 한 번 돌고 끝나면 컨테이너는 "종료" 상태가 됩니다(정상).
> 다시 돌리려면 프로젝트에서 **빌드/시작**을 누르거나 8번 자동화 사용.

---

## 7. 결과 확인
- DSM **File Station → docker/bombam/work/** 폴더를 보세요.
- `봄밤_작품별_최신댓글.xlsx` 가 생성돼 있습니다. (다운로드해서 엑셀로 열기)
- 다른 작업을 돌리고 싶으면 `docker-compose.yml` 의 맨 아래
  `command:` 를 추가하거나 Dockerfile의 `CMD` 를 바꾸세요:
  - 전체수집: `python 1_전체수집.py`
  - 최근30일: `python 3_최근30일_댓글.py`

---

## 8. 자동(주기) 실행 — 선택
매일/매주 자동으로 돌리려면:
1. DSM → **제어판 → 작업 스케줄러**
2. **생성 → 예약된 작업 → 사용자 정의 스크립트**
3. 일정(예: 매일 오전 9시) 설정
4. "작업 설정" 탭의 스크립트에 입력:
   ```bash
   cd /volume1/docker/bombam && /usr/local/bin/docker compose up
   ```
   (경로의 `/usr/local/bin/docker` 는 DSM 버전에 따라 `docker` 만으로도 됨)

## 자주 막히는 곳
- **빌드 실패(네트워크/메모리)**: 메모리 작은 모델이면 빌드 중 멈출 수 있음 →
  `docker-compose.yml` 의 `mem_limit` 를 올리거나, 맥에서 이미지를 만들어 옮기는 방법(고급)을 쓰세요.
- **로그인/Cloudflare로 0건 수집**: 5번 쿠키가 profile 폴더에 제대로 들어갔는지 확인.
- **armv7(32비트) 모델**: Chromium 미지원이라 이 방식 불가.

막히면 **어느 단계의 어떤 화면/에러**인지 알려주세요. 거기에 맞춰 도와드립니다.
