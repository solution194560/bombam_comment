# 테스트 결과

## 통과 정적 검증: 문법·구조
실행 명령:
```bash
python3 -m py_compile news_grok.py browser_lock.py
LC_ALL=C grep -n '[^ -~]' .dockerignore
sed -n '1p' news_grok.py browser_lock.py
```

실제 관찰 결과:
- `py_compile_exit=0`
- `.dockerignore`에서 비-ASCII 문자 검출 없음
- 신규 Python 파일 첫 줄 한국어 헤더 주석 확인:
  - `news_grok.py`: `# Grok 웹 자동화 봇 ...`
  - `browser_lock.py`: `# 프로세스 간 브라우저 동시 실행 방지 락 ...`
- `grok_prompt.txt`는 헤더 검사 제외 조건에 맞게 제외

## 통과 컨테이너 실제 구동
실행 명령:
```bash
docker stop bombam_test 2>&1 || true
docker rm bombam_test 2>&1 || true
rm -f local_test/profile/Singleton*
# 표준/은 빌드 동안 임시로 프로젝트 밖 이동 후 복원
docker build -t bombam:test .
docker run -d --name bombam_test --shm-size=1gb \
  -e HEADLESS=0 \
  --env-file local_test/telegram_test.env \
  -v "$(pwd)/local_test/profile:/app/.browser_data_dir" \
  -v "$(pwd)/local_test/work:/app/out" \
  bombam:test
docker logs --tail 200 bombam_test
docker stop bombam_test
docker rm bombam_test
```

실제 관찰 결과:
- 빌드 성공
- 컨테이너 로그에서 기동 확인:
  - `[news] === AI 뉴스봇 시작 ===`
  - `=== bombam bot 컨테이너 시작 ===`
  - `[grok] === Grok 뉴스봇 시작 ===`
- Grok 자동 루프는 오늘 페이지가 이미 있어 브라우저 구동 전 스킵:
  - `[grok] 오늘 페이지 이미 존재 — 스킵 (Grok news_2026-07-09)`
- 테스트봇 env 적용 확인 및 텔레그램 전송 성공:
  - `telegram_notify_ok: True`
- 테스트 컨테이너 정리 완료: `docker stop bombam_test`, `docker rm bombam_test`
- `표준/` 폴더 복원 확인

- 실제 grok.com 로그인 세션 검증은 사용자 조건의 “실접속 대안” 범위로 대체 검증

## 정적확인 grok 자동화
실행 명령:
```bash
nl -ba news_grok.py | sed -n '51,66p'
nl -ba news_grok.py | sed -n '210,254p'
grep -n "TODO(probe)" news_grok.py
```

실제 관찰 결과:
- 셀렉터 후보는 `textarea`, `contenteditable`, `aria-label`, `placeholder`, `article`, `data-testid` 기반이며 CSS-in-JS 해시 클래스 의존 없음
- `_submit_prompt()`는 입력창 미발견 시 `RuntimeError` 발생
- `_is_logged_in()`은 입력창 미발견을 `False`로 반환하고 상위에서 실패 알림 처리
- `TODO(probe)` 주석이 셀렉터·완료 감지·전송·복사 버튼 위치에 존재
