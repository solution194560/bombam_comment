## 보강 검증 (1차 판정 '수정 필요' — 검증 공백 3건 실증)

1차 판정(pipeline-judge)에서 코드 결함이 아닌 **검증 공백** 3건이 지적됐다. 코드 수정 없이
실제 실행으로 근거를 캡처했다(로컬 Docker `bombam_test`, 실 Notion, 테스트 봇).

### 보강-A — `run_report()` 성공 분기를 본체로 실제 실행 (기존 시나리오 1은 수동 조립이었음)

`news_bot.datetime`을 07-08로 위장(monkeypatch)해 `run_report()`가 오늘을 2026-07-08로 인식하게 하고
**함수 본체를 그대로 호출**. 수동 조립이 아니라 `run_report()`의 성공 분기(탐색→추출→전송→로그)가 완주.

```
[notion] 동일 제목 2개 중복 — 최신 편집본 선택
[news] 리포트 중계 완료 — 조각 1개, 페이지 397a0b7e-561a-81cb-9d53-ecfbd1ab1cf0
```

`리포트 중계 완료` 로그는 `send_long` 성공 뒤에만 찍히는 run_report 마지막 줄 → 실 텔레그램 전송 확인. **[통과]**

### 보강-B — 스케줄 트리거 → last_run_date 기록 → 재실행 없음 (기존엔 '없음' 경로만 관측)

`news_settings.json`을 `daily_time`=컨테이너 시각+1분, `last_run_date`=""로 두고 컨테이너 재시작.
main 루프가 daily_time 도달 시 `run_report()`를 **자동 트리거**(오늘 07-09라 '없음' 경로 — 스케줄 메커니즘 검증).

```
=== 트리거 감지 (40초 경) ===
[news] 오늘 리포트 없음 — 알림 후 종료
# news_settings.json → {"daily_time":"00:18","last_run_date":"2026-07-09"}   (실행 시작 직후 기록)
# 30초 추가 경과 후 '오늘 리포트 없음' 발생 횟수 = 1  (>= 비교 + last_run_date로 재실행 없음)
```

성공 분기 자체는 보강-A에서, 스케줄 트리거·중복방지는 여기서 각각 검증됨. **[통과]**

### 보강-C — 프로세스 kill → 30초 후 자동 재기동 (기존 시나리오 5-1 건너뜀)
