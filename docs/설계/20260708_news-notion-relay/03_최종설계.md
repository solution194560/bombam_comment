## 1. 목표와 성공 기준

### 1.1 무엇을, 왜

현재 `news_bot.py`는 매일 `daily_time`에 4개 LLM(Grok/GPT/GLM/Claude)을 병렬 호출해
뉴스를 조사·종합한 뒤 텔레그램으로 전문을 분할 전송한다. 리포트는 이제 **별도 경로로
이미 Notion에 게시**되므로 생성 파이프라인을 전부 제거하고, 봇의 역할을 다음으로 바꾼다.

매일 `daily_time`(기본 10:00, Asia/Seoul)에

1. Notion API로 'AI NEWs' 부모 페이지(ID `396a0b7e561a80edbb31e60dcc86148a`)의
   **직속** child page 목록을 조회한다(실측으로 통합 보고서가 직속 자식임을 확인 — 2단계 탐색 불필요).
2. 제목에 `통합 보고서_{오늘 YYYY-MM-DD}`가 포함된 페이지를 찾는다(중복 시 최신 편집본).
3. 있으면 그 페이지 블록을 읽어 '핵심 요약(Executive Summary)' 섹션만 추출하고
   **제목 + 요약 + Notion 페이지 링크**를 소유자에게 텔레그램 전송한다(전문 아님).
4. 없으면 'ℹ️ 오늘 리포트 없음' 알림을 1회 보내고 종료한다(당일 재확인 없음 — 사용자 승인).

### 1.2 성공 기준

- [ ] 오늘 자 통합 보고서가 Notion에 있을 때, 10시 이후 소유자 텔레그램으로
      "제목 + 핵심 요약 + 페이지 링크" 메시지가 도착한다(길면 `[i/n]` 분할).
- [ ] 같은 제목이 중복 존재할 때 `last_edited_time` 최신본이 선택되고 로그가 남는다.
- [ ] 오늘 자 보고서가 없을 때 ℹ️ 알림이 정확히 1회 오고, 당일 중복 실행이 없다.
- [ ] Notion 토큰 미설정·오류(401/403/404/429/5xx·529/타임아웃) 시 상황별 문구로
      소유자에게 실패 알림이 온다.
- [ ] `news_llm.py`가 저장소에서 사라지고, `bot.py`(댓글봇)와 기존 스케줄/재기동
      구조는 아무 영향이 없다.
- [ ] 외부 패키지 0개(urllib 등 표준 라이브러리만), 로컬 Docker 테스트 전 시나리오 통과.

## 3. 상세 설계

### 3.1 신규 파일 `news_notion.py`

첫 줄 한국어 역할 주석(코딩 수칙).
`# Notion API 호출 모듈 — 부모 페이지의 child page 탐색·블록 읽기·핵심 요약 추출 (urllib만 사용)`

#### 3.1.1 공통 상수·헤더

```python
NOTION_API = "https://api.notion.com/v1"
NOTION_VERSION = "2022-06-28"   # 안정 버전 확정 — 시나리오 6에서 실호출로 응답 형태 검증
TIMEOUT = 30                     # urlopen 타임아웃(초)
```

모든 요청 헤더는 3개다.

```
Authorization: Bearer {token}
Notion-Version: 2022-06-28
Content-Type: application/json
```

#### 3.1.2 `_get_json(url: str, token: str) -> dict`

- `urllib.request.Request(url, headers=..., method="GET")` → `urlopen(req, timeout=TIMEOUT)`
  → JSON 파싱해 dict 반환.
- **HTTPError 처리** — 응답 본문 앞 500자를 포함한
  `RuntimeError(f"HTTP {e.code} — {본문[:500]}")`를 던진다. 예외 클래스 신설 금지 —
  호출자(`run_report`)가 메시지 접두부(`"HTTP 401"` 등)로 판별한다(§3.2.4).
- **일시 장애 재시도(검토 반영)** — 상태코드가 **429 또는 {500, 502, 503, 504, 529}**이면
  응답의 `Retry-After` 헤더(초, `e.headers.get("Retry-After")`)를 읽어 `min(값, 30)`초 대기 후
  **같은 요청을 1회만 재시도**한다. 헤더가 없거나 숫자가 아니면 2초 대기.
  재시도도 실패하면 예외 전파. (하루 십수 요청이라 실제로 걸릴 일은 드묾 — 최소 방어.)
- 그 외 상태코드(401/403/404 등)는 재시도 없이 즉시 예외.
- 타임아웃·네트워크 오류(`URLError`, `TimeoutError`)는 그대로 전파 —
  `run_report`의 최상위 try/except가 실패 알림으로 변환한다.

#### 3.1.3 `list_children(block_id: str, token: str) -> list`

페이지네이션(100개씩)을 감춘 조회 함수. **부모 목록 조회와 리포트 본문 조회 둘 다
이 함수 하나로** 처리한다(같은 엔드포인트).

```python
base = f"{NOTION_API}/blocks/{block_id}/children"
results = []
cursor = None
for _ in range(10):                      # 안전 상한 10페이지(블록 1,000개)
    params = {"page_size": 100}
    if cursor:
        params["start_cursor"] = cursor
    root = _get_json(base + "?" + urllib.parse.urlencode(params), token)
    results += root.get("results", [])
    if not root.get("has_more"):
        return results
    cursor = root.get("next_cursor")
    if not cursor:
        return results
raise RuntimeError("Notion pagination limit exceeded")   # 10페이지 초과 — 조용한 반환 금지
```

- **(검토 반영)** 커서는 `urllib.parse.quote` 직접 결합이 아니라
  `urllib.parse.urlencode`로 쿼리 전체를 구성한다(커서의 `/` 등 안전 인코딩).
- **(검토 반영)** 10페이지 초과 시 조용히 부분 결과를 반환하지 않고
  `RuntimeError("Notion pagination limit exceeded")`를 던진다 — "보고서 없음"이나
  요약 누락으로 오진되는 것을 막는다. 이 예외는 `run_report`의 포괄 except가
  "내부 오류: Notion pagination limit exceeded" 실패 알림으로 변환한다.
- 중첩 블록(toggle 내부 등)은 **재귀 조회하지 않는다** — 실측으로 리포트가 최상위
  heading_2 + paragraph 평면 구조임을 확인(§0 지적 4, §4.3 생산자 계약).

#### 3.1.4 `find_today_report(parent_id: str, token: str, needle: str) -> tuple | None`

- `list_children(parent_id, token)` 결과에서 `block["type"] == "child_page"`인 블록만 본다.
- 제목은 `block["child_page"]["title"]`.
- **매칭 규칙** — `needle`(= `f"통합 보고서_{오늘 YYYY-MM-DD}"`)이 제목에
  **부분 문자열로 포함**되면 매치(`needle in title`). 이모지(`📊 …`)·앞뒤 장식이 있어도
  안전하고, 날짜가 통째로 들어가므로 다른 날짜와 오매치 불가. 페이지 icon은 title과
  별개 속성이라 애초에 비교 대상 아님.
- **(검토 반영 — 중복 처리 확정)** 매치가 2개 이상이면 각 블록의 **`last_edited_time`
  (블록 공통 필드, ISO 8601 문자열)이 가장 최신(문자열 최대값)인 것을 선택**하고,
  `_log`가 없으므로 `print(f"[notion] 동일 제목 {n}개 중복 — 최신 편집본 선택", flush=True)`
  형식으로 중복 감지 로그를 남긴다. ISO 8601 UTC(`2026-07-08T01:23:45.000Z`) 형식은
  사전순 비교가 시간순 비교와 일치하므로 파싱 없이 `max(..., key=...)`로 충분하다.
  (실측으로 현재 `📊 통합 보고서_2026-07-08` 중복 2개가 실존 — 가상 방어가 아님.)
- 매치되면 `(block["id"], title)` 반환, 없으면 `None`.

#### 3.1.5 `rich_text_to_plain(rich_text: list) -> str`

- `"".join(rt.get("plain_text", "") for rt in rich_text)` — 각 항목의 `plain_text` 연결.
- 링크·볼드 등 서식은 전부 무시(전문은 Notion 링크로 본다).

#### 3.1.6 `extract_summary(blocks: list) -> str`

'핵심 요약(Executive Summary)' 섹션 추출. 순수 함수(네트워크 없음 → 단독 테스트 쉬움).

**1차 규칙(정상 경로)**

1. 최상위 블록을 순서대로 훑어 `heading_1`/`heading_2`/`heading_3` 중
   `rich_text_to_plain(...)` 결과에 `"핵심 요약"`이 포함되거나 소문자 변환 문자열에
   `"executive summary"`가 포함되는 **첫 heading**을 시작점으로 삼는다.
   heading 레벨(1/2/3)을 기억한다.
2. 그다음 블록부터, **레벨이 시작 heading과 같거나 더 높은(숫자가 같거나 작은)
   heading을 만나기 전까지** 아래 타입의 텍스트를 모은다.
   - `paragraph` → 본문 그대로 한 줄
   - `bulleted_list_item` / `numbered_list_item` → 앞에 `"• "`를 붙여 한 줄
   - `quote` → 본문 그대로 한 줄
   - 그 외 타입(table, divider, image, callout 등)은 조용히 건너뛴다.
3. 빈 문자열 블록은 버리고, 모은 줄들을 `"\n"`으로 연결해 반환한다.
   길이 제한 없음 — `send_long`이 3,500자 단위로 분할한다.

**폴백 규칙**

- **폴백 1** — heading을 못 찾았거나 모은 텍스트가 공백뿐이면, 최상위 블록 맨 앞부터
  `paragraph`/`bulleted_list_item`/`numbered_list_item`/`quote` 텍스트를 순서대로 모아
  **최대 1,000자**(초과 시 마지막 줄바꿈에서 절단 + `"…(이하 생략 — 전문은 링크 참고)"` 부착)
  반환. 첫 줄에 `"(핵심 요약 섹션을 찾지 못해 본문 앞부분을 표시합니다)"` 안내 부착.
- **폴백 2** — 폴백 1도 빈 문자열이면
  `"(요약을 추출하지 못했습니다 — 전문은 아래 링크에서 확인하세요)"` 반환.
- 어느 경로든 **빈 문자열을 반환하지 않는다** → 호출자는 분기 없이 그대로 메시지에 넣는다.

#### 3.1.7 `page_url(block_id: str) -> str`

- `"https://www.notion.so/" + block_id.replace("-", "")` — child_page 블록 id(UUID)에서
  대시를 제거한 32자 hex가 곧 페이지 주소(Notion 공식 URL 규칙).

#### 3.2.3 새 `run_report()` — 최종 의사코드

부모 페이지 ID는 `news_bot.py` 상수(설정 스키마 확장 없음 — 최소 구현).

```python
NOTION_PARENT_PAGE_ID = "396a0b7e561a80edbb31e60dcc86148a"  # 'AI NEWs' 페이지 (직속 자식 탐색 — 실측 확인)

def run_report() -> None:
    now = datetime.now()                        # 컨테이너 TZ=Asia/Seoul
    today = now.strftime("%Y-%m-%d")
    today_disp = f"{today} ({_WEEKDAY_KR[now.weekday()]})"

    try:
        keys = load_api_keys()

        # 1) 토큰 없으면 외부 호출 전에 즉시 실패 알림
        if not keys.get("notion"):
            _notify_failure(today_disp,
                "Notion 토큰 없음 — work/api_keys.json에 notion 키를 입력하면 "
                "다음 날부터 동작합니다.")
            return

        # 2) 오늘 자 통합 보고서 child page 탐색 (중복 시 최신 편집본)
        found = news_notion.find_today_report(
            NOTION_PARENT_PAGE_ID, keys["notion"], f"통합 보고서_{today}")

        # 3) 없으면 ℹ️ 알림 1회 후 종료 (last_run_date는 main()이 기록 → 당일 재시도 없음, 사용자 승인)
        if not found:
            send_long(f"ℹ️ 오늘({today_disp}) 통합 보고서가 Notion에 아직 없습니다.\n"
                      "게시되면 내일 시각부터 자동 확인합니다.", _owner_chat_id())
            _log("오늘 리포트 없음 — 알림 후 종료")
            return

        page_id, title = found

        # 4) 리포트 페이지 블록 읽기 → 핵심 요약 추출
        blocks = news_notion.list_children(page_id, keys["notion"])
        summary = news_notion.extract_summary(blocks)   # 항상 비어있지 않은 문자열
        url = news_notion.page_url(page_id)

        # 5) 전송 — 제목 + 요약 + 링크 (전문 아님)
        msg = f"{title}\n\n{summary}\n\n📎 전문 보기(Notion)\n{url}"
        n = send_long(msg, _owner_chat_id())
        _log(f"리포트 중계 완료 — 조각 {n}개, 페이지 {page_id}")

    except Exception as e:
        # 어떤 예외에도 소유자에게 실패 알림 (기존 이중 방어 구조 유지)
        _log("run_report 내부 오류:\n" + traceback.format_exc())
        _notify_failure(today_disp, _failure_hint(e))
```

`title`은 child_page 제목 원문(이모지 포함)을 메시지 첫 줄에 그대로 쓴다.
`notify_telegram`은 `disable_web_page_preview=true`라 링크 미리보기가 없다(기존 동작).

#### 3.2.4 실패 알림 분기 — `_failure_hint(e) -> str` (확정 에러 분기 표)

`_get_json`이 던지는 `RuntimeError("HTTP {code} — {본문}")` 메시지의 **접두부**로 판별하는
작은 헬퍼 함수를 `news_bot.py`에 둔다(예외 클래스 신설 없음). 최종 알림은 기존
`_notify_failure` 형식(`⚠️ AI 뉴스 리포트 실패 ({today_disp})\n{message}`) 그대로.

| 상황 | 판별 | message |
|---|---|---|
| 토큰 미설정 | `keys["notion"]` 빈 값 (예외 아님 — §3.2.3의 1번 분기) | `Notion 토큰 없음 — work/api_keys.json에 notion 키를 입력하면 다음 날부터 동작합니다.` |
| HTTP 401 | `msg.startswith("HTTP 401")` | `Notion 인증 실패(401) — 토큰이 잘못됐거나 만료됐습니다. api_keys.json의 notion 키를 확인하세요.` |
| HTTP 403 | `msg.startswith("HTTP 403")` | `Notion 권한 부족(403) — 통합(integration)에 읽기 권한이 없습니다. Notion 통합 설정에서 콘텐츠 읽기 권한을 확인하세요.` |
| HTTP 404 | `msg.startswith("HTTP 404")` | `Notion 페이지 접근 불가(404) — 페이지가 없거나 통합(integration)에 공유되지 않았습니다. Notion에서 'AI NEWs' 페이지 → 연결(Connections)에 통합을 추가하세요.` (Notion은 공유 안 된 페이지도 404를 돌려줌) |
| HTTP 429 (1회 재시도 후에도) | `msg.startswith("HTTP 429")` | `Notion 요청 제한(429) — 잠시 후 재시도해도 실패했습니다. 내일 자동 재시도됩니다.` |
| HTTP 5xx·529 (1회 재시도 후에도) | `msg.startswith("HTTP 5")` (500/502/503/504/529 포괄) | `Notion 서버 일시 장애({코드}) — 재시도해도 실패했습니다. 내일 자동 재시도됩니다.` |
| 타임아웃/네트워크/기타 | 위에 해당 없음 | `내부 오류: {예외 첫 줄 or 타입명}` (기존 포괄 경로 — 페이지네이션 상한 예외도 여기로) |

판별 순서 주의 — `"HTTP 5"` 접두부 검사는 429보다 뒤에 두든 앞에 두든 무방하나
(429는 "HTTP 4"), 400번대 개별 검사(401/403/404/429)를 먼저 나열하고 마지막에
`"HTTP 5"` 포괄을 두는 순서로 구현한다.

### 3.3 설정/스키마 — `api_keys.json`

`OUTPUT_DIR/api_keys.json`(컨테이너 `/app/out/`, NAS `work/`, 로컬 `local_test/work/`)의
기대 스키마 변경. **파일 위치·자동 생성 로직·gitignore는 무변경.**

```json
{
  "notion": ""
}
```

(`DEFAULT_KEYS = {"notion": ""}`)

- **마이그레이션 불필요** — `load_api_keys()`는 `DEFAULT_KEYS`의 키만 읽으므로 기존
  4키 파일에 `"notion"` 항목만 추가하면 된다. 남은 LLM 키는 무시될 뿐 오류 아님.
  파일이 없으면 새 스키마로 자동 생성 + `[news] api_keys.json 생성됨 — work 폴더에서
  키를 채워 주세요` 로그(기존 동작).
- **(검토 반영)** 더 이상 쓰지 않는 실제 LLM 키는 보안상 사용자가 파일에서 지우는 편이
  좋다 — 배포 후 사용자에게 안내한다(코드는 강제하지 않음).
- Notion 토큰은 **읽기 전용 internal integration**으로 발급하고, Notion에서 'AI NEWs'
  페이지 → 연결(Connections)에 통합을 추가해야 하위 페이지까지 읽힌다.
- `news_settings.json`은 스키마 무변경(`daily_time`, `last_run_date`).

### 4.3 Notion API 특유 주의점 + 생산자 계약

- **공유 안 된 페이지는 403이 아니라 404** — 404 문구에 "통합에 공유" 안내(§3.2.4).
  403은 통합 자체의 읽기 권한 부족(restricted_resource)일 때 온다 — 별도 문구.
- **`Notion-Version` 헤더 누락 시 400** — 모든 요청에 고정값 `2022-06-28` 첨부.
- **child_page 제목과 페이지 icon은 별개** — 매칭은 title 문자열만, icon(📊)은 무시.
- **커서는 불투명 값** — `urlencode`로 쿼리 구성(직접 문자열 결합 금지).
- **rate limit 평균 3req/s** — 하루 십수 요청이라 여유 크지만 429 방어는 둠.
- **생산자 계약(문서화 — 검토 지적 4 대응)** — 통합 보고서 페이지는
  **"핵심 요약 heading(제목에 '핵심 요약' 또는 'Executive Summary' 포함)과 그 본문을
  최상위 블록에 둔다"**를 게시 측 규칙으로 유지한다. 실측(2026-07-08 보고서)으로
  현재 구조가 이 계약을 만족함을 확인했다. 계약이 깨지면 폴백 1로 본문 앞 1,000자가
  나간다(링크는 항상 전달 — 완전 실패 없음).

## 6. 구현 시 주의사항

### "이것만 지키면 됨" 체크리스트

1. **탐색은 1단계** — `AI NEWs`(ID `396a0b7e561a80edbb31e60dcc86148a`) **직속** 자식에서
   `child_page` 타입 + 제목에 `통합 보고서_{YYYY-MM-DD}` 부분 문자열 포함으로 찾는다.
   날짜 폴더 하위로 내려가지 않는다(실측 근거 있음).
2. **중복 매치는 `last_edited_time` 최신본** — 문자열 `max` 비교로 충분. 중복이면 로그 1줄.
3. **커서는 `urllib.parse.urlencode`** — 문자열 직접 결합 금지. **10페이지 초과는
   `RuntimeError("Notion pagination limit exceeded")`** — 조용한 부분 반환 금지.
4. **재시도는 429·500·502·503·504·529만, 딱 1회** — `Retry-After` 헤더(없으면 2초,
   상한 30초). 401/403/404는 즉시 예외.
5. **에러 판별은 `RuntimeError("HTTP {code} — …")` 접두부 문자열** — 예외 클래스
   신설 금지. 분기 문구는 §3.2.4 표의 문구를 **그대로** 쓴다.
6. **`Notion-Version: 2022-06-28` 고정** — 모든 요청 헤더 3종(Authorization/버전/Content-Type).
7. **`extract_summary`는 빈 문자열을 절대 반환하지 않는다** — 폴백 1(앞 1,000자) →
   폴백 2(링크 안내 문구). 최상위 블록만 읽는다(재귀 없음).
8. **`main()` 루프·`load_news_settings`·`send_long`·`_notify_failure`·`_owner_chat_id`는
   한 글자도 바꾸지 않는다**(`>=` 비교와 `last_run_date` 선기록이 함정 8번 대응이다).
9. **`DEFAULT_KEYS = {"notion": ""}` 하나만** — `load_api_keys()` 로직 무변경.
10. **삭제는 §3.2.1 표의 항목만** — `news_llm.py` 파일 삭제 + import/threading 고아 정리.
    notify.py, bot.py, entrypoint.sh, Dockerfile, docker-compose.yml은 손대지 않는다.
11. **새 소스 파일 첫 줄에 한국어 역할 주석**, news_bot.py 헤더 주석·docstring 갱신.
12. **로컬 테스트는 반드시 `--env-file local_test/telegram_test.env`** — 없으면 운영 봇과
    409 충돌. 전 시나리오(0~8) 통과 전에는 push·NAS 배포 금지.

### 기타

- `.claude/settings.local.json`에 `news_llm.py` 관련 허용 항목이 남아 있어도 런타임
  무관 — 이번 커밋에서 건드리지 않는다(후속 정리 대상으로만 기록, 검토 지적 8).
- 뉴스봇에는 수동 `/run`이 없다 — `last_run_date` 선기록과 충돌하는 경로 없음.
- 커밋은 논리 단위로 제안한다. 예. "feat: 뉴스봇을 Notion 통합 보고서 중계로 전환
  (LLM 파이프라인 제거)" 1건이면 충분하다(news_llm.py 삭제 포함 — 같은 논리 단위).
