# 설계: AI 뉴스봇의 Notion→텔레그램 중계 전환

- 작성일: 2026-07-08
- 모드: 정식(구조/다중 파일/스키마 변경)
- 대상 브랜치 기준: 현재 작업본 (`news_bot.py`, `news_llm.py`, `notify.py`, `entrypoint.sh` 확인 완료)

## 1. 목표와 배경

### 1.1 무엇을, 왜

현재 `news_bot.py`는 매일 `daily_time`에 4개 LLM(Grok/GPT/GLM/Claude)을 병렬 호출해
뉴스를 조사하고 Claude로 종합한 뒤 텔레그램으로 평문 분할 전송한다. 이제 리포트는
**별도 경로로 이미 Notion에 게시**되므로, 봇에서 생성 파이프라인(조사·종합·프롬프트·절단)을
전부 제거하고, 봇의 역할을 아래처럼 바꾼다.

매일 `daily_time`(기본 10:00, Asia/Seoul)에

1. Notion API(응용 프로그램끼리 데이터를 주고받는 규약)로 'AI NEWs' 부모 페이지
   (ID `396a0b7e561a80edbb31e60dcc86148a`)의 하위 child page(자식 페이지) 목록을 조회한다.
2. 제목에 `통합 보고서_{오늘 YYYY-MM-DD}`가 포함된 페이지를 찾는다.
3. 있으면 그 페이지의 블록을 읽어 '핵심 요약(Executive Summary)' 섹션만 추출하고,
   **요약 텍스트 + Notion 페이지 링크**를 소유자에게 텔레그램 전송한다(전문 아님).
4. 없으면 '오늘 리포트 없음' 알림을 1회 보내고 종료한다.

### 1.2 성공 기준 (이것이 되면 완료)

- [ ] 오늘 자 통합 보고서가 Notion에 있을 때, 10시 이후 소유자 텔레그램으로
      "제목 + 핵심 요약 + 페이지 링크" 메시지가 도착한다(길면 `[i/n]` 분할).
- [ ] 오늘 자 보고서가 없을 때 '없음' 알림이 정확히 1회 오고, 당일 중복 실행이 없다.
- [ ] Notion 토큰 미설정·오류(401/404/429/타임아웃) 시 소유자에게 실패 알림이 온다.
- [ ] 4개 LLM 호출 코드(`news_llm.py` 등)가 저장소에서 사라지고, `bot.py`(댓글봇)와
      기존 스케줄/재기동 구조는 아무 영향이 없다.
- [ ] 외부 패키지 0개(urllib 등 표준 라이브러리만), 로컬 Docker 테스트 전 시나리오 통과.

## 3. 상세 설계

### 3.1 신규 파일: `news_notion.py`

첫 줄에 한국어 역할 주석을 단다(코딩 수칙).
`# Notion API 호출 모듈 — 부모 페이지의 child page 탐색·블록 읽기·핵심 요약 추출 (urllib만 사용)`

#### 3.1.1 공통 상수·헤더

```python
NOTION_API = "https://api.notion.com/v1"
NOTION_VERSION = "2022-06-28"   # 안정 버전 — blocks children API는 이 버전으로 충분
TIMEOUT = 30                     # urlopen 타임아웃(초). 조회성 GET이라 짧게
```

모든 요청 헤더는 다음 3개다.

```
Authorization: Bearer {token}
Notion-Version: 2022-06-28
Content-Type: application/json
```

(GET 요청이라 Content-Type은 필수가 아니지만 무해하므로 통일해 넣는다.)

#### 3.1.2 `_get_json(url: str, token: str) -> dict`

- `urllib.request.Request(url, headers=..., method="GET")` → `urlopen(req, timeout=TIMEOUT)`
  → JSON 파싱해 dict 반환.
- **HTTPError 처리** — `news_llm.py`의 `_post_json`과 같은 패턴으로, 응답 본문 앞 500자를
  포함한 `RuntimeError("HTTP {code} — {본문}")`를 던진다. 상태코드별 의미는 호출자
  메시지에 반영한다(§3.2.4).
- **429(요청 과다) 처리** — 응답의 `Retry-After` 헤더(초)를 읽어 `min(값, 30)`초 대기 후
  **같은 요청을 1회만 재시도**한다. 헤더가 없으면 2초 대기. 재시도도 429/실패면 예외 전파.
  (하루 요청 수가 많아야 십수 회라 실제로는 걸릴 일이 거의 없다 — 최소 방어만.)
- 타임아웃·네트워크 오류(`URLError`, `TimeoutError`)는 그대로 예외 전파 —
  호출자(`run_report`)의 최상위 try/except가 실패 알림으로 변환한다.

#### 3.1.3 `list_children(block_id: str, token: str) -> list`

페이지네이션(긴 목록을 100개씩 나눠 받는 방식)을 감춘 조회 함수.

```
url = f"{NOTION_API}/blocks/{block_id}/children?page_size=100"
results = []
cursor = None
for _ in range(10):                      # 안전 상한 10페이지(블록 1,000개)
    u = url + (f"&start_cursor={urllib.parse.quote(cursor)}" if cursor else "")
    root = _get_json(u, token)
    results += root.get("results", [])
    if not root.get("has_more"):
        break
    cursor = root.get("next_cursor")
    if not cursor:
        break
return results
```

- `has_more`가 참이면 `next_cursor`를 `start_cursor` 쿼리 파라미터로 넘겨 다음 100개를
  받는다. 커서 문자열은 `urllib.parse.quote`로 URL 인코딩한다.
- **부모 목록 조회와 리포트 본문 조회 둘 다 이 함수 하나로** 처리한다(같은 엔드포인트).
- 중첩 블록(toggle 내부 등)은 **재귀 조회하지 않는다** — 최상위 블록만 읽는 최소 구현.
  실제 리포트 구조상 핵심 요약은 최상위 heading + paragraph로 확인됨(리스크는 §6 참고).

#### 3.1.4 `find_today_report(parent_id: str, token: str, needle: str) -> tuple | None`

- `list_children(parent_id, token)` 결과에서 `block["type"] == "child_page"`인 블록만 본다.
- 제목은 `block["child_page"]["title"]`에서 얻는다.
- **매칭 규칙(확정)** — `needle`(= `f"통합 보고서_{오늘 YYYY-MM-DD}"`)이 제목 문자열에
  **부분 문자열로 포함**되면 매치. 즉 `"통합 보고서_2026-07-08" in title`.
  - 이렇게 하면 제목 안의 이모지(`📊 통합 보고서_...`)·앞뒤 공백·장식 문구가 있어도
    안전하게 매치되고, 페이지 icon(제목과 별개 속성)은 애초에 비교 대상이 아니다.
  - 날짜가 통째로 들어가므로 다른 날짜 보고서와 오매치될 수 없다.
  - 여러 개가 매치되면 **첫 번째**(Notion 페이지 내 배치 순서상 먼저 오는 것)를 쓴다.
- 매치되면 `(block["id"], title)` 반환, 없으면 `None`.

#### 3.1.5 `rich_text_to_plain(rich_text: list) -> str`

- Notion 블록의 `rich_text` 배열에서 각 항목의 `plain_text`를 그대로 이어붙인다.
  `"".join(rt.get("plain_text", "") for rt in rich_text)`
- 링크·볼드 등 서식은 전부 무시한다(최소 구현 — 요약문에 하이퍼링크가 필요하면
  본문 전문을 Notion 링크로 보게 하면 된다).

#### 3.1.6 `extract_summary(blocks: list) -> str`

'핵심 요약(Executive Summary)' 섹션 추출. 순수 함수(네트워크 없음 → 단독 테스트 쉬움).

**1차 규칙(정상 경로)**

1. 최상위 블록을 순서대로 훑어 `heading_1`/`heading_2`/`heading_3` 중
   `rich_text_to_plain(...)` 결과에 `"핵심 요약"` 이 포함되거나, 소문자로 바꾼 문자열에
   `"executive summary"` 가 포함되는 **첫 heading**을 시작점으로 삼는다.
   heading 레벨(1/2/3)을 기억한다.
2. 그다음 블록부터, **레벨이 시작 heading과 같거나 더 높은(숫자가 같거나 작은)
   heading을 만나기 전까지** 아래 타입의 텍스트를 모은다.
   - `paragraph` → 본문 그대로 한 줄
   - `bulleted_list_item` / `numbered_list_item` → 앞에 `"• "` 를 붙여 한 줄
     (번호 재현은 하지 않는다 — 최소 구현)
   - `quote` → 본문 그대로 한 줄
   - 그 외 타입(table, divider, image, callout 등)은 **조용히 건너뛴다**
     (표는 텔레그램에 안 맞으므로 요구사항대로 제외).
3. 빈 문자열이 되는 블록은 버리고, 모은 줄들을 `"\n"`으로 연결해 반환한다.
   길이 제한은 두지 않는다 — 어차피 `send_long`이 3,500자 단위로 분할한다.

**폴백(대비책) 규칙 — 명확히 확정**

- **폴백 1** — 위 heading을 못 찾았거나 찾았지만 모은 텍스트가 공백뿐이면,
  페이지 최상위 블록 맨 앞에서부터 `paragraph`/`bulleted_list_item`/`numbered_list_item`/
  `quote` 텍스트를 순서대로 모아 **최대 1,000자**(초과 시 마지막 줄바꿈에서 절단하고
  `"…(이하 생략 — 전문은 링크 참고)"` 부착)를 반환한다. 반환문 첫 줄에
  `"(핵심 요약 섹션을 찾지 못해 본문 앞부분을 표시합니다)"` 안내를 붙인다.
- **폴백 2** — 폴백 1도 빈 문자열이면(텍스트 블록이 전혀 없음)
  `"(요약을 추출하지 못했습니다 — 전문은 아래 링크에서 확인하세요)"` 를 반환한다.
- 어느 경로든 **빈 문자열을 반환하지 않는다** → 호출자는 분기 없이 그대로 메시지에 넣는다.

#### 3.1.7 `page_url(block_id: str) -> str`

- `"https://www.notion.so/" + block_id.replace("-", "")`
- child_page 블록의 id(UUID, 대시 포함 36자)에서 대시를 제거한 32자 hex가 곧 페이지
  주소가 된다(Notion의 공식 URL 규칙).

#### 3.2.3 새 `run_report()` — 의사코드

부모 페이지 ID는 **`news_bot.py` 상수**로 둔다(설정 파일 스키마 확장 없음 — 최소 구현.
바뀔 일이 생기면 그때 설정으로 승격).

```python
NOTION_PARENT_PAGE_ID = "396a0b7e561a80edbb31e60dcc86148a"  # 'AI NEWs' 페이지

def run_report() -> None:
    now = datetime.now()                        # 컨테이너 TZ=Asia/Seoul
    today = now.strftime("%Y-%m-%d")
    today_disp = f"{today} ({_WEEKDAY_KR[now.weekday()]})"

    try:
        keys = load_api_keys()

        # 1) 토큰 없으면 외부 호출 전에 즉시 실패 알림 (기존 anthropic 검사와 같은 패턴)
        if not keys.get("notion"):
            _notify_failure(today_disp,
                "Notion 토큰 없음 — work/api_keys.json에 notion 키를 입력하면 "
                "다음 날부터 동작합니다.")
            return

        # 2) 오늘 자 통합 보고서 child page 탐색
        found = news_notion.find_today_report(
            NOTION_PARENT_PAGE_ID, keys["notion"], f"통합 보고서_{today}")

        # 3) 없으면 '없음' 알림 1회 후 종료 (last_run_date는 main()이 이미 기록 → 당일 재시도 없음)
        if not found:
            send_long(f"ℹ️ 오늘({today_disp}) 통합 보고서가 Notion에 아직 없습니다.\n"
                      "게시되면 내일 시각부터 자동 확인합니다.", _owner_chat_id())
            _log("오늘 리포트 없음 — 알림 후 종료")
            return

        page_id, title = found

        # 4) 리포트 페이지 블록 읽기 → 핵심 요약 추출
        blocks = news_notion.list_children(page_id, keys["notion"])
        summary = news_notion.extract_summary(blocks)   # 항상 비어있지 않은 문자열
        url = news_notion.page_url(page_id)

        # 5) 전송 — 제목 + 요약 + 링크 (전문 아님)
        msg = f"{title}\n\n{summary}\n\n📎 전문 보기(Notion)\n{url}"
        n = send_long(msg, _owner_chat_id())
        _log(f"리포트 중계 완료 — 조각 {n}개, 페이지 {page_id}")

    except Exception as e:
        # 어떤 예외에도 소유자에게 실패 알림 (기존 이중 방어 구조 유지)
        _log("run_report 내부 오류:\n" + traceback.format_exc())
        _notify_failure(today_disp,
            f"내부 오류: {str(e).splitlines()[0] if str(e) else type(e).__name__}")
```

- `title`은 child_page 제목 원문(이모지 포함)을 그대로 메시지 첫 줄에 쓴다 —
  별도 헤더를 만들지 않는다(최소 구현).
- `notify_telegram`이 `disable_web_page_preview=true`로 보내므로 링크 미리보기가
  메시지를 어지럽히지 않는다(기존 동작, 무변경).

#### 3.2.4 실패 알림 문구 (상황별)

`_notify_failure(today_disp, message)` → `⚠️ AI 뉴스 리포트 실패 ({today_disp})\n{message}`
형식은 기존 그대로. `message`는 HTTPError 코드로 분기해 초보자용 원인 힌트를 덧붙인다.

| 상황 | message |
|---|---|
| 토큰 미설정 | `Notion 토큰 없음 — work/api_keys.json에 notion 키를 입력하면 다음 날부터 동작합니다.` |
| HTTP 401 | `Notion 인증 실패(401) — 토큰이 잘못됐거나 만료됐습니다. api_keys.json의 notion 키를 확인하세요.` |
| HTTP 404 | `Notion 페이지 접근 불가(404) — 페이지가 없거나 통합(integration)에 공유되지 않았습니다. Notion에서 'AI NEWs' 페이지 → 연결(Connections)에 통합을 추가하세요.` (Notion은 공유 안 된 페이지도 404를 돌려준다) |
| HTTP 429 (재시도 후에도) | `Notion 요청 제한(429) — 잠시 후 다시 시도해도 실패했습니다. 내일 자동 재시도됩니다.` |
| 타임아웃/네트워크 | `내부 오류: ...` (기존 포괄 경로 — 예외 첫 줄 포함) |

구현 시 분기는 `_get_json`이 던지는 `RuntimeError("HTTP 401 — ...")` 문자열의 코드로
단순 판별하면 충분하다(예외 클래스 신설 금지 — 최소 구현).

### 3.3 설정/스키마 변경 — `api_keys.json`

`OUTPUT_DIR/api_keys.json`(컨테이너 `/app/out/`, NAS `work/`, 로컬 `local_test/work/`)의
기대 스키마가 바뀐다. **파일 위치·자동 생성 로직·gitignore는 무변경.**

이전.

```json

  "xai": "",
  "openai": "",
  "zai": "",
  "anthropic": ""
}
```

이후 (`DEFAULT_KEYS = {"notion": ""}`).

```json

}
```

- **마이그레이션 불필요** — `load_api_keys()`는 `DEFAULT_KEYS`에 있는 키만 읽으므로,
  NAS·로컬에 남아 있는 기존 4키 파일에 `"notion"` 항목만 추가하면 된다. LLM 키가
  남아 있어도 무시될 뿐 오류가 아니다. 파일이 아예 없으면 새 스키마로 자동 생성되고
  `[news] api_keys.json 생성됨 — work 폴더에서 키를 채워 주세요` 로그가 남는다(기존 동작).
- Notion 토큰은 **읽기 전용 internal integration**으로 발급하고, Notion에서 'AI NEWs'
  페이지에 해당 통합을 연결(Connections 추가)해야 하위 페이지까지 읽을 수 있다.
- `news_settings.json`은 스키마 무변경(`daily_time`, `last_run_date`).

### 4.3 Notion API 특유 주의점

- **공유 안 된 페이지는 403이 아니라 404** — 실패 문구에 "통합에 공유" 안내를 넣는다(§3.2.4).
- **`Notion-Version` 헤더 누락 시 400** — 모든 요청에 고정값 `2022-06-28` 첨부.
- **child_page 제목과 페이지 icon은 별개** — 매칭은 title 문자열만 사용, icon(📊)은 무시.
- **next_cursor는 URL-safe가 아닐 수 있음** — `urllib.parse.quote`로 인코딩(§3.1.3).
- **rate limit 평균 3req/s** — 이 봇은 하루 십수 요청이라 여유가 크지만 429 방어는 둠.
