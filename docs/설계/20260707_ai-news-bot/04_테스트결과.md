# 테스트 결과

이 결과는 1차 테스트(시나리오 7 실패 후 조기 중단)를 대체하는 2차 전체 재실행 결과다

정적 검증을 먼저 수행했다. `python3 ast.parse`로 `news_bot.py`, `news_llm.py`, `bot.py`, `notify.py`, `ridi_collector.py` 문법 확인 통과. `load_news_settings()`에서 깨진 JSON 예외 경로의 `corrected = True`, `"aa:bb"` 및 `"8:00"` 교정 경로도 코드 리뷰로 확인했다.

주의: 첫 컨테이너 기동 때 기존 `local_test/work/news_settings.json` 상태 때문에 `Anthropic 키 없음` 경로가 한 번 즉시 진입했다. 이후 컨테이너는 `TELEGRAM_BOT_TOKEN=` 및 `TELEGRAM_CHAT_ID=` 빈 값으로 재기동했고, 시나리오 검증은 `notify_telegram`/`send_long` 몽키패치로 캡처했다.

## [통과] 시나리오 1. 두 프로세스 공존 + 파일 자동 생성

수행 명령

```bash
docker stop bombam_test 2>&1; docker rm bombam_test 2>&1
find local_test/profile -maxdepth 1 -name 'Singleton*' -print -delete
docker build -t bombam:test .
docker run -d --name bombam_test --shm-size=1gb \
  -e HEADLESS=0 \
  -e TELEGRAM_BOT_TOKEN= \
  -e TELEGRAM_CHAT_ID= \
  -v "$(pwd)/local_test/profile:/app/.browser_data_dir" \
  -v "$(pwd)/local_test/work:/app/out" \
  bombam:test sleep infinity
docker logs bombam_test --tail 50
```

실제 관찰 결과

- `HEADLESS=0`로 기동했다.
- `bot.py`는 기동하지 않았다. CMD는 `sleep infinity`로 오버라이드했다.
- `/proc` 기준 프로세스는 `bash /app/entrypoint.sh sleep infinity`, `sleep infinity`, `python -u /app/news_bot.py`.
- 로그: `[news] === AI 뉴스봇 시작 === daily_time=23:59 키보유: 없음`.
- `news_settings.json`, `api_keys.json` 스키마 존재 확인.
- `/status`는 `bot.py 절대 기동 금지` 조건 때문에 수행하지 않았다.

## [통과] 시나리오 2. 키 전무(초기 상태)

수행 명령

```bash
docker exec -i bombam_test python -
```

몽키패치 내용: `news_llm.call_grok`, `call_openai`, `call_zai`, `call_claude_research`, `call_claude_synthesize`를 호출되면 실패하도록 대체. `send_long`은 캡처 함수로 대체.

실제 관찰 결과

- `api_keys.json` 전부 빈 값.
- 스케줄러 동작을 모의해 `last_run_date`를 오늘 `2026-07-07`로 기록한 뒤 `run_report()` 호출.
- LLM 제공자 함수 호출 0회.
- 캡처된 알림: `Anthropic 키 없음 — 종합 단계 수행 불가`.
- `last_run_date`는 `2026-07-07`로 저장되어 같은 날 재실행 조건이 false.

## [통과] 시나리오 8. 크래시 복구 루프

수행 명령

```bash
docker exec bombam_test sh -lc 'for p in /proc/[0-9]*; do c=$(tr "\000" " " < "$p/cmdline" 2>/dev/null || true); case "$c" in *news_bot.py*) kill ${p#/proc/}; exit 0;; esac; done; exit 1'
sleep 40
docker logs bombam_test --tail 80
```

실제 관찰 결과

- `python -u /app/news_bot.py` PID kill 확인.
- 로그: `[news] 뉴스봇 프로세스 종료 감지 — 30초 후 재기동`.
- 이후 로그: `[news] === AI 뉴스봇 시작 === daily_time=23:59 키보유: 없음`.
- `/proc` 기준 새 `python -u /app/news_bot.py` PID 생존 확인.
- `bot.py`는 계속 미기동.

## [통과] 시나리오 9. docker restart 후 스케줄

수행 명령

```bash
docker restart bombam_test
docker logs --since "$SINCE" bombam_test
# 이후 last_run_date=""로 변경 후 25초 대기
docker logs --since "$SINCE2" bombam_test
cat local_test/work/news_settings.json
```

실제 관찰 결과

- `last_run_date=2026-07-07`, `daily_time=00:00` 상태로 `docker restart`.
- 재시작 직후 로그는 시작 로그만 존재: `[news] === AI 뉴스봇 시작 === daily_time=00:00 키보유: 없음`.
- 같은 날 재실행 로그 없음.
- 이후 `last_run_date=""`로 비우자 다음 루프에서 `Anthropic 키 없음 — 종합 불가로 조사 호출 생략` 로그 발생.
- 파일은 `last_run_date: "2026-07-07"`로 다시 기록됨.
