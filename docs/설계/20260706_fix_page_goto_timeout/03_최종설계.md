# 최종설계: 1순위 자동 재시도 로직

## 핵심 목표
**문제:** 연속 4개 작품 실패로 조기 중단 시, 남은 작품들이 수집되지 않으므로 사람이 `/run` 버튼을 눌러야 함.

**해결:** 조기 중단 후 20~30분 뒤에 자동으로 재시도 (인터넷이 복구될 때까지 기다렸다가 한 번 더 시도)

---

## 변경 파일 목록

| 파일 | 변경 내용 | 영향도 |
|------|---------|--------|
| `4_매일알림.py` | 조기 중단 감지 시 settings.json에 재시도 예약 시간 저장 | 중간 |
| `bot.py` | 메인 폴링 루프에서 재시도 시간 확인 후 자동 실행 | 중간 |
| `ridi_collector.py` | **변경 없음** (이미 stats["aborted"] 반환) | 0 |

**변경 후:**
```python
elif stats["aborted"]:
    # 조기 중단 (일부만 수집됨)
    partial_abort = True
    fail_msg = (f"⚠️ 수집 조기 중단\n\n"
                f"작품 {stats['total']}개 중 {stats['succeeded']}개만 수집됨\n"
                f"(네트워크 문제로 추정)\n\n"
                f"현재까지 수집한 댓글을 이하 전송합니다.")
    
    # 20~30분 뒤 자동 재시도 예약
    from datetime import datetime, timedelta
    retry_at = (datetime.now() + timedelta(minutes=25)).strftime("%Y-%m-%d %H:%M:%S")
    s = load_settings()
    s["retry_at"] = retry_at
    from ridi_collector import save_settings
    save_settings(s)
    print(f"  25분 뒤 자동 재시도 예약: {retry_at}")
```

**설명:**
- 조기 중단이 감지되면 settings.json에 `"retry_at"` 필드 추가
- 값: 현재 시간 + 25분 (20~30분 범위의 중간값)
- 25분은 회선 장애 복구 대기 시간과 다음 폴링 주기를 고려한 값

### 2️⃣ `bot.py`

**위치:** 메인 루프 (`main()` 함수 내, 약 200~250줄 추정)

**추가할 로직:**

메인 루프(약 210~260줄)의 `while True:` 섹션에 다음을 추가:

```python
# [재시도 확인] settings.json의 retry_at이 지났으면 자동 실행
try:
    s = rc.load_settings()
    retry_at_str = s.get("retry_at", "")
    if retry_at_str:
        from datetime import datetime
        retry_at = datetime.strptime(retry_at_str, "%Y-%m-%d %H:%M:%S")
        now = datetime.now()
        if now >= retry_at and not _job_running.is_set():
            print(f"[자동 재시도] 이전 조기 중단 후 {(now - retry_at).total_seconds():.0f}초 경과, 다시 수집 실행")
            # 재시도는 자동(is_auto=True) 아님 → 실패 알림만 전송되고 success 알림은 정상 흐름
            run_daily_job(is_auto=False)
            # 재시도 완료 후 플래그 제거 (다시 설정되지 않는 한 재실행 안 함)
            s.pop("retry_at", None)
            rc.save_settings(s)
except Exception as e:
    print(f"  [재시도 확인 오류] {str(e)[:60]}", flush=True)

# 기존 getUpdates 폴링...
offset = None  # 또는 기존 폴링 로직
```

**위치 선택:**
- 메인 루프의 `while True:` 직후, 가장 먼저 실행
- 또는 `getUpdates` 폴링 전에 실행
- 주기: 약 1초마다 확인 (기존 폴링과 함께)

**설명:**
- 메인 루프가 이미 1초마다 `getUpdates` 폴링 중이므로, 같은 주기로 확인 가능
- retry_at 시간이 지났으면 `run_daily_job(is_auto=False)`로 실행
  - `is_auto=False` → 실패 알림만 전송 (성공 알림은 자동 알림이 아니므로 정상 흐름)
- 완료 후 `retry_at` 제거 (1회만 실행)

## 테스트 계획

### 로컬 Docker 테스트 (필수)

**테스트 시나리오 1: 조기 중단 재현 + 재시도 확인**

1. 로컬 Docker 빌드 및 실행
2. `docker exec bombam_test python3 4_매일알림.py` 실행 (조기 중단 시뮬레이션)
   - 또는 작품 URL을 잘못된 주소로 임시 변경해 실패 유도
3. `work/settings.json`의 `"retry_at"` 필드 생성 확인
4. 25분 대기 (또는 `settings.json`의 `retry_at`을 과거로 수정해 즉시 실행)
5. bot.py 로그에서 `[자동 재시도]` 메시지 확인
6. 두 번째 수집이 정상 진행되는지 확인

**테스트 시나리오 2: 정상 수집 시 재시도 필드 생성 안 됨**

1. 정상 수집 완료
2. settings.json에 `retry_at` 필드가 없음 확인

**테스트 시나리오 3: 재시도 성공 후 플래그 제거**

1. 조기 중단 → `retry_at` 설정
2. 재시도 실행 → `retry_at` 제거 확인
3. 그 이후 다시 대기 (추가 재시도 안 함)

## 리스크 및 주의사항

| 항목 | 설명 |
|------|------|
| **중단 vs 재시도 경합** | 조기 중단 도중 동시에 `/run` 누르거나 정기 알림 시간이 되지 않으려면, `_job_running` 플래그로 보호 필요 (기존 코드에 이미 있음) |
| **타임존** | `datetime.now()` 사용 → 컨테이너 TZ=Asia/Seoul 필수 확인 |
| **settings.json 동시 접근** | bot.py와 4_매일알림.py가 동시에 쓰지 않도록 주의 (현재 아키텍처상 문제 없음: 4_매일알림.py는 subprocess로 독립 실행) |
