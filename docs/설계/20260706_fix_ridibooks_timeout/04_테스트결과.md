## [통과] 시나리오 1 - 정상 동작 회귀
수행 명령:
```bash
cd "/Users/hyundeok/Downloads/봄밤작가 작품 리스트와 댓글 평가-3"
docker stop bombam_test 2>&1
docker rm bombam_test 2>&1
find local_test/profile -maxdepth 1 -name 'Singleton*' -print -delete
docker build -t bombam:test .
docker run -d --name bombam_test --shm-size=1gb \
  -e HEADLESS=0 \
```

## [통과] 시나리오 2 - 리디북스 장애 모의
수행 명령:

## [통과] 시나리오 3 - 텔레그램 장애 모의
수행 명령:
```bash

find local_test/profile -maxdepth 1 -name 'Singleton*' -print -delete
docker run -d --name bombam_test --shm-size=1gb \

  -v "$(pwd)/local_test/work:/app/out" \

```

실제 관찰 결과: 시작 메시지 전송은 `sendMessage 오류: <urlopen error timed out>`로 실패했지만 프로세스는 계속 유지됐다. 이후 `getUpdates`가 연속 실패했고 타임스탬프 기준으로 실패 간격이 `약 61초 → 62초 → 64초`로 늘어나 60초 롱폴링 타임아웃 위에 `1초 → 2초 → 4초` 백오프가 실제 적용됨을 확인했다. 경고 로그도 첫 실패 1회만 출력돼 로그 도배가 발생하지 않았다.
